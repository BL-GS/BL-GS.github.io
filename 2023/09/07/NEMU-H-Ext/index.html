H Extension Difftest

通过 Spike 进行 difftest 功能验证

git clone https://github.com/OpenXiangShan/riscv-isa-sim.git
cd riscv-isa-sim
git checkout h-ext
cd difftest && make -j

即可获得用于 NEMU Difftest 的 Spike 动态链接库

KVM NEMU 模拟

  1. 按照教程 KVM RISCV64 on QEMU · kvm-riscv/howto Wiki (github.com) 构建一个 QEMU 上的 KVM Linux 工作负载。QEMU 输出可以作为 Debug 参照
  2. 在完成 busybox 的配置后（主要是将一个没有 initram 的 Linux 镜像放进去文件系统后），打开 Linux menuconfig 配置
    $ make -C linux O=`pwd`/build-riscv64 menuconfig


General setup ->
    Initramfs source file(s) 填入 busybox 的安装目录地址

Kernel features ->  
     SBI v0.1 support
 
Device Drivers -> 
     Character devices
         Serial driver ->
             Early console using RISC-V SBI
         RISC-V SBI console support
  3. 将 riscv-pk/dts at noop · OpenXiangShan/riscv-pk (github.com) 中的设备树文件下载下来，主要是 system.dts 和 noop.dtsi 文件。配置 platform.dtsi 软连接到 noop.dtsi。
  4. 修改 system.dtsi 中 L6: cpu 的 riscv,isa 为 rv64imafdch 。主要确保 h 标志位。
  5. 修改 noop.dtsi 中的 bootargs 为：console=hvc0 earlycon=sbi。
  6. 记得将 platform.dtsi 中设置的内存大小调大一点，不然会出问题，在之后 Linux 镜像加载出问题时也可以尝试将内存加大一点：
    L11: memory@100000000 {
    device_type = "memory";
    reg = <0x0 0x80000000 0x0 0x20000000>;
};
  7. 整合设备树 dtc -O dtb -o xiangshan.dtb system.dtb。注意 system.dts 和 platform.dtsi 在同一目录下，
  8. 编译 opensbi：
    make PLATFORM=generic CROSS_COMPILE=riscv64-unknown-linux-gnu- FW_FDT_PATH=<dtb path>/xiangshan.dtb FW_PAYLOAD_PATH=<workspace path>/build-riscv64/arch/riscv/boot/Image  -j
  9. 得到的 opensbi/build/platform/generic/fireware/fw_payload.bin 即可作为可执行文件给 NEMU 执行

常见问题

  1. 最简单粗暴的方式：
    1. 删掉重做，能解决绝大部分问题
  2. 重新编译 Linux 的时候，又要在控制台重新配置 config：
    1. 注意是否配置好了 CROSS_COMPILE 和 ARCH，是否删除掉了 Linux 文件
    2. 假如出现该问题，建议完全删掉构建文件夹，重新操作一次，避免种种未知问题
  3. NEMU 在启动之后，输入没有反应
    1. 那可能是因为 NEMU 默认不开启键盘中断。最简单的做法是在 busybox 文件系统的安装目录（如按教程为 __install 目录）下，修改 /etc/init.d/rcS 文件，添加需要执行的操作。如在教程中提到加载 KVM 模块并启动 guest OS 的操作，就可以在 rcS 文件中添加：
      insmod kvm.ko
./apps/lkvm-static run -m 128 -c2 --console serial -p "console=ttyS0 earlycon" -k ./apps/Image --debug
      其他的操作可以参考网上 busybox 以及 initram 的参考资料
  4. 如何配合 NEMU 的检查点
    1. 基于使用 opensbi 的 generic 架构编译的假设，我们修改 opensbi 的 platform/generic/objects.mk 文件后，重新编译即可：
      - FW_TEXT_START=0x80000000
+ FW_TEXT_START=0x800a0000
...
ifeq ($(PLATFORM_RISCV_XLEN), 32)
# This needs to be 4MB aligned for 32-bit system
- FW_PAYLOAD_OFFSET=0x400000
+ FW_PAYLOAD_OFFSET=0x760000
else
# This needs to be 2MB aligned for 64-bit system
- FW_PAYLOAD_OFFSET=0x200000
+ FW_PAYLOAD_OFFSET=0x360000
endif
